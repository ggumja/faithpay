# FaithPay Wireframes (Text-based)

이 문서는 `FaithPay_Functional_Specification.md (v5.0)`를 기반으로 설계된 **신도용 봉헌 페이지**와 **관리자 대시보드**의 화면 구조를 정의합니다.

---

## 1. 텍스트 와이어프레임 (Text-based Wireframes)

디벨로퍼 및 기획자가 화면 구조와 데이터 필드를 한눈에 파악할 수 있는 상세 화면 설계입니다.

### 1.1 신도용: 봉헌 입력 화면 (Donation Input)

```text
+------------------------------------------+
| [로고] OO교회 / OO사찰 / OO성당           |
+------------------------------------------+
| [ Step 1: 봉헌 종류 선택 ]                |
| +--------------------------------------+ |
| | ( ) 십일조 / 보시 / 교무금 (V)         | |
| | ( ) 감사헌금 / 인등접수 / 미사예물     | |
| | ( ) 건축헌금 / 불사공양 / 특별봉헌     | |
| | ( ) 기타 (직정입력)                   | |
+------------------------------------------+
| [ Step 2: 신원 및 정성 입력 ]             |
| 성명: [ 길동홍 ]  전화번호: [ 010-1234- ] |
|                                          |
| (종교별 특화 필드 - 선택 시 노출)          |
| [기독교] 기도제목: [ 가족 건강 기원... ]   |
| [불교]   발원문/사주: [ 乙巳年 ...     ]  |
| [천주교] 세례명: [ 프란치스코 ]            |
+------------------------------------------+
| [ Step 3: 금액 설정 ]                     |
| [ 1만 ] [ 3만 ] [ 5만 ] [ 10만 ] [ 직접 ] |
| 금액: [ 50,000 ] 원                      |
+------------------------------------------+
| [ Step 4: 정기 봉헌 여부 ]                |
| [ ] 매월 정기적으로 봉헌합니다 (구독)     |
|     결제일: ( ) 5일  ( ) 15일  ( ) 25일  |
+------------------------------------------+
| [ 다음 단계로 (결제하기) ]                 |
+------------------------------------------+
```

## 2. 사용자 흐름 (User Flow)

1.  **접속 (QR/Link):** 해당 단체의 전용 슬러그(`faithpay.com/church-a`) 또는 도메인으로 접속.
2.  **항목 선택:** 봉헌할 항목(예: 주일 헌금) 선택 및 금액 입력.
3.  **정보 입력:** 종교별 특화 필드(기도제목 등) 및 개인정보 입력(비회원 가능).
4.  **결제:** PG 연동을 통한 신용카드/간편결제/가상계좌 진행.
5.  **완료:** 디지털 영수증 발급 및 카카오 알림톡 전송. 관리자 대시보드에 실시간 반영.
